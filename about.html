It’s a Small World is a project based on stupidness. The stupidness of how easily the world is interconnected, and how stupidly I can connect myself to some of the most famous people on Earth through just a handful of connections. The theory of six degrees of separation is also affirmed in this project, with not a single connection out of the 309 ever stretching onto the 7th degree of connection.

Code citations
d3.forceSimulation
This is the core of D3 that runs all the 'physics', it's essentially an engine that continuously calculates where every one of the 309 nodes should be based on the forces applied to them.
Observable. d3-force simulation. D3 by Observable, 2023. https://d3js.org/d3-force/simulation.
d3.forceLink
This creates a sort of spring force along each edge, so connected nodes get pulled toward each other. I could control how strong and how far, which is what the Link Force and Link Distance sliders adjust.
Observable. d3-force link. D3 by Observable, 2023. https://d3js.org/d3-force/link.
d3.forceManyBody
This is basically the opposite of forceLink, every node pushes every other node away from it. Without this all 309 nodes would just collapse into the centre. The Repel Force slider controls how strong this is.
Observable. d3-force many-body. D3 by Observable, 2023. https://d3js.org/d3-force/many-body.
d3.forceX / d3.forceY
These two forces work together to pull all nodes toward the centre of the screen so the graph doesn't drift off to the side. They're what the Centre Force slider adjusts.
Observable. d3-force position. D3 by Observable, 2023. https://d3js.org/d3-force/position.
d3.forceCollide
This one prevents nodes from sitting directly on top of each other. It treats every node as a circle with a certain radius and makes sure no two overlap, which matters a lot with 309 people in one graph.
Observable. d3-force collide. D3 by Observable, 2023. https://d3js.org/d3-force/collide.
simulation.stop() / simulation.tick()
I found that without this, the graph would start from zero every time the Start button was clicked and all the nodes would fly around chaotically. So I stop the sim, manually run 500 steps behind the scenes while the intro screen covers it, then reveal it already settled.
Observable. d3-force simulation.tick. D3 by Observable, 2023. https://d3js.org/d3-force/simulation#simulation_tick.
simulation.alpha()
Alpha is basically how much energy the simulation has. When it's high nodes move fast and chaotically, when it's near zero everything has settled. I restart at 0.15 after the pre-ticks so things gently drift to their final positions.
Observable. d3-force simulation.alpha. D3 by Observable, 2023. https://d3js.org/d3-force/simulation#simulation_alpha.
d3.scaleSqrt
I didn't want the most connected nodes to be gigantic, but I did want them to clearly look bigger. The square root scale does this — it maps connection count to radius in a way that feels proportional without Jimmy Fallon's node being the size of the screen.
Observable. d3-scale scaleSqrt. D3 by Observable, 2023. https://d3js.org/d3-scale#scaleSqrt.
d3.zoom
This is what lets you zoom in and drag the graph around. The transform gets applied to a wrapper group so everything moves together — nodes, edges, and labels all shift as one.
Observable. d3-zoom. D3 by Observable, 2023. https://d3js.org/d3-zoom.
d3.drag
This lets you grab and move nodes around the canvas. While you're dragging, the node gets temporarily pinned so it stays where your mouse is. When you release, it goes back to being pulled around by physics.
Observable. d3-drag. D3 by Observable, 2023. https://d3js.org/d3-drag.
selection.join()
This is how D3 connects your data to the actual elements drawn on screen. For every edge in my data array there's one SVG line, for every node there's one SVG group. It handles creating, updating, and removing them automatically.
Observable. d3-selection joining. D3 by Observable, 2023. https://d3js.org/d3-selection/joining.
d3.zoomIdentity
When the graph first appears it needs to be at a zoom level where you can see everything. This creates the starting transform so it opens centered and fits within the window.
Observable. d3-zoom zoomIdentity. D3 by Observable, 2023. https://d3js.org/d3-zoom#zoomIdentity.
new Map()
I use JavaScript Maps a lot in this project. The main places are: making sure the Epstein group overrides work by deduplicating nodes with the last entry winning, building the connection lookup for the hover tooltip, and counting how many edges each person has to size their node.
MDN Web Docs. "Map." Mozilla, 2024. https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Map.
BFS (breadth-first search)
BFS is how I calculated the degrees of separation for every person on the graph. It starts from my node and fans outward, recording how many steps it takes to reach each person. That's what shows up in the hover tooltip.
Wikipedia. "Breadth-first search." Accessed March 2026. https://en.wikipedia.org/wiki/Breadth-first_search.
clip-path animation
This is the circular shrinking animation when you click Start. I found this transition on transition.style and adapted it. The trick is that the graph is already rendered underneath during the animation, so when it finishes there's no loading delay.
MDN Web Docs. "clip-path." Mozilla, 2024. https://developer.mozilla.org/en-US/docs/Web/CSS/clip-path.
CSS Grid
The Key panel on the graph uses a two-column grid layout. I used CSS Grid for this instead of manually positioning things.
MDN Web Docs. "CSS Grid Layout." Mozilla, 2024. https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_grid_layout.
element.classList
classList is basically how I control the state of elements by adding or removing CSS classes. I use it all over the site — the collapsible panels, the Show Relationships button changing state, and toggling edge labels on and off.
MDN Web Docs. "Element.classList." Mozilla, 2024. https://developer.mozilla.org/en-US/docs/Web/API/Element/classList.
XMLSerializer
The print versions have a Save SVG button. XMLSerializer is how I convert the live SVG element on the page into a downloadable string so you can export it.
MDN Web Docs. "XMLSerializer." Mozilla, 2024. https://developer.mozilla.org/en-US/docs/Web/API/XMLSerializer.
position: fixed
All the UI overlays on the graph use position fixed so they stay on screen while you're panning and zooming the graph behind them. Without this, the panels would scroll away with the graph.
MDN Web Docs. "position." Mozilla, 2024. https://developer.mozilla.org/en-US/docs/Web/CSS/position.
CSS @font-face
Both fonts load via @font-face. Lombard is only available as a local file, and I also added a local fallback for Manrope in case the Google Fonts CDN is down or unavailable.
MDN Web Docs. "@font-face." Mozilla, 2024. https://developer.mozilla.org/en-US/docs/Web/CSS/@font-face.
CSS clamp()
clamp() lets me set a font size that scales with the viewport but never goes below or above certain limits. The intro title uses this so it looks right on both a phone and a large monitor without me having to write separate media queries.
MDN Web Docs. "clamp()." Mozilla, 2024. https://developer.mozilla.org/en-US/docs/Web/CSS/clamp.
Audio.play()
There's a short sound that plays when you click Start. The .catch() at the end is because some browsers block audio autoplay and would throw an error — this just silently ignores it if that happens.
MDN Web Docs. "HTMLAudioElement." Mozilla, 2024. https://developer.mozilla.org/en-US/docs/Web/API/HTMLAudioElement.
setTimeout
After the animation plays, the intro screen is technically still there even though you can't see it. I use setTimeout to remove it entirely after 3.1 seconds so it doesn't accidentally block clicks and hovers on the graph.
MDN Web Docs. "setTimeout." Mozilla, 2024. https://developer.mozilla.org/en-US/docs/Web/API/setTimeout.
input[type="range"]
The Forces panel has four sliders, one for each major physics parameter. Each one listens for the input event which fires on every movement, not just on release, so the graph responds in real time as you drag.
MDN Web Docs. "input type range." Mozilla, 2024. https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input/range.
d.fx / d.fy
In D3's force simulation, you can pin a node in place by setting fx and fy. My node is always pinned to the exact centre of the screen so all the other nodes orbit around me, which is kind of the point.
Observable. d3-force simulation nodes. D3 by Observable, 2023. https://d3js.org/d3-force/simulation#simulation_nodes.
mouseover / mouseout
mouseover and mouseout are what trigger the hover interaction. When you hover a node, everything not connected to it dims, connected edges highlight, and the tooltip appears. When you leave, it all resets.
MDN Web Docs. "mouseover event." Mozilla, 2024. https://developer.mozilla.org/en-US/docs/Web/API/Element/mouseover_event.
@media (max-width: 600px)
This media query only applies on screens narrower than 600px. It shrinks the UI panels and adjusts the initial zoom so the graph fills the phone screen instead of appearing as a tiny dot in the middle.
MDN Web Docs. "CSS media queries." Mozilla, 2024. https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_media_queries.
will-change: clip-path
This hints to the browser that the clip-path property is about to animate, so it can move that work to the GPU in advance. Without it the intro animation could stutter on slower devices.
MDN Web Docs. "will-change." Mozilla, 2024. https://developer.mozilla.org/en-US/docs/Web/CSS/will-change.
text-anchor: middle
Every name label in the graph uses text-anchor: middle so it centres over its node. Without this, shorter names would look off-centre compared to longer ones.
MDN Web Docs. "text-anchor." Mozilla, 2024. https://developer.mozilla.org/en-US/docs/Web/SVG/Attribute/text-anchor.
stroke-opacity
When you hover a node, its connected edges go to full opacity and the rest drop to nearly invisible. stroke-opacity is how I control this per-edge in SVG.
MDN Web Docs. "stroke-opacity." Mozilla, 2024. https://developer.mozilla.org/en-US/docs/Web/SVG/Attribute/stroke-opacity.
